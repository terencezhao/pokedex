A Web Service for Cataloging Pokemon

For ProWebservice, I created a webservice for managing pokemon. The Pokedex web service allows users to create, read, update, and delete Pokemon. Pokemon have a "Many-to-One" relationship with an Element type (Fire, Water, Grass, etc.) It uses the Spring Boot framework to expose endpoints for CRUD operations from the "/pokedex" endpoint. There is also a "/api" enpoint that can be used to retrieve pokemon content as JSON. ThymeLeaf was used to dynamically populate HTML and visualize the data stored within the in memory H2 database. The pages were created and styled with BootStrap.
